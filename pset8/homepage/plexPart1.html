There is no place like 127.0.0.1, part 1

by Hein ReynekeJuly 30th, 2018

Salutations readers, I have been doing some really cool stuff at work in the development department and it is quite revolutionary. But enough about my work project, that I might be telling you all about later on in a later post, stay tuned.

So after putting in some thought and got a plan together of what I would like to do for the tech/dev tutorial part of this blog, I came up with a multipart project that will serve multiple functions as well as learn while we are doing it.

First we will need to decide what platform (OS) we are going to use for the project, although you can use any one of the 3 big ones, we will be using Ubuntu (Linux) for this project. This is my OS of choice, MacOS comes second as I use that for work and most of my daily tasks, lastly Windows only for gaming purposes though.

Many people might think Linux is such a difficult OS to use but in fact it is quite easy, you just need get used to it moving from Windows. Think about the first time you worked on Windows, everything was new and daunting, read  Windows 3.1.

We are going to build a media server with automated tools to rename your Movies and Series that you own as well as stream the digital version to multiple devices from your server, we can also use it to store photos and music and stream that across devices as well.

After that we are going to build on it further with all the tools needed to start our first development project and what options we have, but we will get to that later on. We will also be using this box as a remote proxy that can be used if you are connected on a slow connections and need your http download to be downloaded faster than what you have access to at the time and lastly we will use it as a jump box to gain access to your home network.

OK lets get down to business:

1. Downloading the needed software

We need to download the OS, for this project we will be using Ubuntu Server.

Next we can use VM (virtual machine) software like virtualbox that is totally free and available for all major operating systems or you could put it on a stand alone pc / server. I am personally running a Linux server at home that runs 24/7 that I use for many things.

2. Setting up the VM

Once downloaded and installed, launch virtualbox and click on the new button on the top left, select Linux as the Type, Ubuntu as the version and give it a name that is identifiable, in my case ill call it Lazegeek Tutorial

Now we need to set the Memory Size to 4Gb, we can always increase this at a later stage if we need to, and click continue.

Next we need to allocate it hard disk space, select “Create a virtual hard disk” now and click create. In the new window choose “VDI”, click continue, choose “Dynamically allocated”, click continue.

Then set the hard disk size to at least 20Gb and click create, we can also add more later on if need be. You will also need to take into account if you are going to use the VM as a Media Server then you need to add more hard disk space, in my opinion you should rather use a stand alone pc / server for that.

Thats it! Now we have the first makings of our VM that we will be using.

3. Installing Ubuntu on our new VM

First we need to mount the Ubuntu iso that we just downloaded, to do this click on settings, go to storage and then select the cd drive (tile says “Empty”) under the “controller: IDE” section. On the right hand side you need to click the “CD” icon and then choose your iso file that you downloaded.

Now its time to start up your VM, give it a few moments to start, once you get to the language selection screen choose your preferred language. Leave the next few screens (preferred keyboard layout, Network Connections, Proxy Address - unless you are working behind a proxy, if you don’t know what this is you are probably not & Ubuntu Archive Mirror) on its defaults as the installer is pretty good at picking these settings for itself. For this tutorial we are going to select “Entire Disk” when you get to the “Filesystem Setup Menu” and choose the newly created hard disk.

Once selected the installer will automatically setup the necessary partitions for you so no need to worry about that, so we just choose “Done” and c confirm that you want to continue. On the next screen you need to fill in details for your Ubuntu Setup, see example below.

On the next screen we do not select anything and just choose “Done” and then “Reboot Now”. It ill display a message “Please remove the installation media, then press” Enter”, to remove the iso click “Devices”, hover over “Optical Drives” and then click “Remove disk from virtual drive” and press enter so that the installer can continue.

Congratulation, you have successfully installed Ubuntu (version 18.04 as of this writing), see anyone can do it, it is very easy, it just seems daunting but it really isn’t. Next time we will be installing our media center software and some other tools to get things automated.

Let me know in the comments what you guys think of the tutorial and if you have any suggestions.

May the force be with you!
